Hello World! - A First Look

We will begin our exploration of JActor2 with a program, HelloWorld, that prints "Hello World!". In the process, we will encounter a number of classes which play a major role when using JActor. 

import org.agilewiki.jactor2.core.blades.misc.Printer;
import org.agilewiki.jactor2.core.facilities.Facility;
import org.agilewiki.jactor2.core.messages.SyncRequest;

public class HelloWorld {
    public static void main(final String[] _args) throws Exception {
        Facility facility = new Facility();
        try {
            Printer printer = Printer.stdoutAReq(facility).call();
            SyncRequest<Void> printRequest = Printer.stdoutSReq(facility).call();
            printRequest.call();
        } finally {
            facility.close();
        }
    }
}

Facility 

        Facility facility = new Facility();
        try {
            ...
        } finally {
            facility.close();
        }

A facility provides a thread pool and other resources, with a default thread pool size of 20. Facilities are used by reactors, with every reactor having a reference to one facility. A program can have 1 or more facilities, and does not exit normally until all the facilities are closed.

Printer 

            Printer printer = Printer.stdoutAReq(facility).call();

A printer is an IsolationBlade, which is to say an application object that uses an IsolationReactor. It prints to a PrintStream, System.out being the default.

The static method Printer.stdoutAReq returns an AsyncRequest which, when called, returns the facility's printer. And if the facility has no printer, one is safely created.

SyncRequest<RESPONSE_TYPE> 

            SyncRequest printRequest = printer.printlnSReq("Hello World!");

Blades, like printer, expose requests which can be used to invoke various operations, SyncRequest being a type of request whose implementation uses a return statement to return their result. In this case, printRequest is a request to print "Hello World!".

call 

            printRequest.call();

Here we print "Hello World!". But it is printed from the printer's own thread, not from the main thread.

The call method of a request provides one way of invoking a request. The printRequest.call method sends the request to the blade's thread and blocks the main thread until a result is received. The call method is the only method in JActor which blocks a thread. (The call method can not be used by a blade, as it must be invoked from a non-facility thread.)
